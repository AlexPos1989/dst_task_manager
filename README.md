# Трекер Боссов Don't Starve Together

Веб-приложение для отслеживания вашего прогресса в победе над боссами в *Don't Starve Together*.

## Возможности
- **Полный список боссов**: Включает более 35 боссов, взятых с [Don't Starve Wiki](https://dontstarve.fandom.com/wiki/Category:Boss_Monsters).
- **Визуальное отслеживание**: Большие, качественные изображения каждого босса.
- **Интерактивный прогресс**: Нажмите на любого босса, чтобы отметить его как "убитого" с помощью четкого визуального индикатора (красный крест и оттенки серого).
- **Постоянное хранение**: Ваш прогресс автоматически сохраняется в локальном хранилище браузера, поэтому вы не потеряете его, если закроете вкладку.
- **Адаптивный дизайн**: Отлично работает на настольных и мобильных устройствах.
- **Темный режим**: Тематически соответствует эстетике игры.

## Технологический стек
- **React 19**: Для UI компонентов и управления состоянием.
- **Tailwind CSS 4**: Для стилизации и адаптивной верстки.
- **Vite**: Для быстрой разработки и сборки.
- **Motion**: Для плавной анимации.
- **Lucide React**: Для иконок.

## Локальный запуск и развертывание

### Предварительные требования
- **Node.js**: Убедитесь, что у вас установлен Node.js (версии 18 или выше).
- **npm**: Пакетный менеджер Node.js (обычно устанавливается вместе с Node.js).

### Установка
1. Скачайте или клонируйте репозиторий проекта.
2. Откройте терминал в корневой папке проекта.
3. Установите зависимости командой:
   ```bash
   npm install
   ```

### Режим разработки
Для запуска в режиме разработки с горячей перезагрузкой (HMR):
```bash
npm run dev
```
Сервер запустится на `http://localhost:3000`. В этом режиме сервер (`server.ts`) использует Vite middleware для обработки запросов к фронтенду.

### Сборка и запуск (Production)
Для запуска в режиме, приближенном к боевому (или для развертывания на хостинге):

1. **Соберите фронтенд**:
   Эта команда создаст оптимизированную версию сайта в папке `dist`.
   ```bash
   npm run build
   ```

2. **Запустите сервер**:
   Эта команда запустит `server.ts`, который будет раздавать статические файлы из папки `dist` и обрабатывать API запросы.
   ```bash
   npm start
   ```
   Сайт будет доступен по адресу `http://localhost:3000`.

### Структура и База Данных
- **Сервер (`server.ts`)**: Это единая точка входа. Он запускает Express сервер, который обслуживает API (`/api/*`) и статические файлы фронтенда.
- **База Данных (`bosses.db`)**: Это файл SQLite базы данных.
  - Он создается автоматически в корневой папке проекта при первом запуске сервера, если его нет.
  - Вам не нужно создавать его вручную.
  - Если вы переносите проект на другой компьютер/хостинг и хотите сохранить прогресс, скопируйте файл `bosses.db` вместе с остальными файлами.

### Файлы для хостинга
Если вы загружаете проект на VPS или хостинг с поддержкой Node.js, вам понадобятся:
- `package.json` и `package-lock.json`
- `server.ts`
- Папка `dist` (после выполнения `npm run build` локально, либо выполните сборку на сервере)
- Папка `public` (если есть статические ресурсы, не попавшие в бандл, например `slash.mp3`)
- `bosses.db` (опционально, если хотите перенести существующие данные)

На хостинге выполните `npm install --production` (или просто `npm install`) и запустите `npm start`.
